Styli a great fit for React-like framework or library, Let's get started with [React](https://reactjs.org/).

## Installation

```bash
npm i @styli/core @styli/react
```

Besides `@styli/core`, we also need React binding package `@styli/react`.

## Use in component

Now we're going to build a user profile card ui with `View`,`Image`,`Text`.

```jsx title="src/components/ProfileCard.ts"
import React from 'react'
import { View, Text, Image } from '@styli/react'

export default () => {
  return (
    <View toCenterY p-10 w-260 bgWhite rounded-10 shadowMD>
      <Image circle-48 src="/img/jobs.jpg" />
      <View ml-10>
        <View f-20 fontBold>
          Steve Jobs
        </View>
        <Text gray800>Co-founder of Apple Inc.</Text>
      </View>
    </View>
  )
}
```

The code above will build the following UI:

```jsx live
<View bgGray10 p6>
  <View toCenterY p-10 w-260 bgWhite rounded-10 shadowMD>
    <Image circle-48 src="/img/jobs.jpg" />
    <View ml-10>
      <View f-20 fontBold>
        Steve Jobs
      </View>
      <Text gray800>Co-founder of Apple Inc.</Text>
    </View>
  </View>
</View>
```

As you can see, we can build a more complex interface with few code. At first glance, you might say: "Isn't this a disguised inline style?" This is indeed similar to the inline style. They both write the style on JSX, but maybe this is the only similarity of them. The way to writing styles of Styli, I call it "Atomic props".

Compared with inline style, "Atomic props" has many advantages:

- **Less Code**, inline styles has a larger amount of code, which can easily make JSX messy, but "Atomic props" retains the simplicity of JSX and is more readable.

- **Style reuse**, using inline styles, it is easy to quickly expand the code, using "Atomic props", the code size will not expand too much as the project grows.

## @styli/react

The code in `@styli/react` is very simple, there is no black magic in it, but simply wraps the html element with `styled`.

```jsx title="@styli/react source code"
import { styled } from '@styli/styled'

export const View = styled('div')
export const Text = styled('span')
export const Image = styled('img')
export const Anchor = styled('a')
export const Heading = styled('h2')
```

Note that when customizing the Styli component, you need to use the `styled` Api in `@styli/styled`.

You can view the source code of `@styli/react` directly in Github：[Souce code](https://github.com/forsigner/styli/blob/master/packages/react/src/index.tsx)。

We can see that `@styli/react` only provides `View`, `Text`, `Image`, `Anchor`, and `Heading` components. During the development process, we recommend encapsulating your own components according to your needs.

For example, if you like the style of html elements, you can encapsulate a set of components like this:

```jsx title="@styli/react 源码"
import { styled } from '@styli/styled'

export const H1 = styled('h1')
export const H2 = styled('h2')
export const H3 = styled('h3')
export const P = styled('p')
export const Div = styled('div')
export const Span = styled('span')
export const Img = styled('img')
export const Button = styled('button')
export const Input = styled('input')

// more....
```

## Use in HTML elements

Atomic Props can also be used directly in html elements. In other words, you can directly add Atomic Props to html tags, the code is similar to the following:

```jsx
<div p6 f5 bgRed200>
  Use styli in HTML tag
</div>
```

To use Atomic Props in html elements, there are two ways to use them:

- Babel Preset
- JSX Pragma

### Babel Preset

The first way is to add a babel preset `@styli/babel-preset-styli` to the project by configuring babel.

```json title=".babelrc"
{
  "presets": ["@styli/babel-preset-styli"]
}
```

Then, you can use Atomic Props in HTML elements:

```jsx
import React from 'react'

export default () => {
  return (
    <div p6 f5 bgRed200>
      Use styli with Babel Preset
    </div>
  )
}
```

Note that if you want to get TypeScript type hints, you need to extend the attribute types of HTML by creating a new `d.ts` file in the project root directory, you can name it `index.d.ts`, the code is as follows:

```jsx title="index.d.ts"
import { AtomicProps } from '@styli/types';

declare module 'react' {
  interface HTMLAttributes<T> extends AtomicProps {}
}

```

![hint](/img/Intellisense.png)

### JSX Pragma

If you cannot control the babel configuration of the project, you can use the **JSX Pragma** instead:

```jsx
/** @jsx jsx */
import { jsx } from '@styli/react'

export default () => {
  return (
    <div p6 f5 bgOrange20>
      Use styli with JSX Pragma
    </div>
  )
}
```

Why the components above can use Atomic Props in HTML directly? It is because the following statement of `JSX Pragma` is added at the beginning of the component.

```jsx
/** @jsx jsx */
import { jsx } from '@styli/react'
```
