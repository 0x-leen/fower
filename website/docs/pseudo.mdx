Each Atomic Props of Styli supports CSS pseudo-classes. The format of the Atomic Props that enable Pseudo-classes style is: `{prop}--{pseudo}`, for example: `bgRed400--hover`, `bgYellow500--active`, `opacity-70--disabled`, `w-20--visited`, etc.

## Hover

```jsx live
<Button bgRed300 bgRed400--hover red800 p4 rounded-10 border="none">
  Hover me
</Button>
```

## Active

```jsx live
<Button bgYellow300 bgYellow500--active yellow800 p4 rounded-10 border="none">
  Click Me
</Button>
```

## Disabled

```jsx live
<View toCenter>
  <Button bgRed300 red800 p4 rounded-10 border="none" mr-20>
    Normal
  </Button>
  <Button disabled bgRed300 opacity-70--disabled red800 p4 rounded-10 border="none">
    Disabled
  </Button>
</View>
```

## Focus

```jsx live
<Input bgGray200 outlineNone bgWhite--focus p2 border borderRed400--focus />
```

## Child

```jsx live
<View>
  <View red400--first-child>One</View>
  <View>Two</View>
  <View green400--last-child>Three</View>
</View>
```

## 支持的伪类

Styli 支持下面所有伪类：

- active
- checked
- disabled
- enabled
- default
- empty
- focus
- focus-within
- invalid
- hover
- link
- visited
- first-child
- last-child
- after
- before
- placeholder
- selection

## About file size

Each Atomic Props in Styli supports CSS pseudo-classes and 18 commonly used pseudo-classes. You may have questions about whether this will cause the code size of the Styli package to explode. The answer is no.

In traditional CSS, if each CSS class supports Pseudo-classes, a large amount of CSS code is required, because CSS is not programmable, and it can only accumulate code through enumeration.

Thanks to the flexibility of JS, Styli only needs dozens of lines of code to make all Atomic Props support Pseudo-classes. If you are interested in how to Styli Pseudo plugin, you can take a look at [styli-plugin-pseudo](https://github.com/forsigner/styli/blob/master/packages/styli-plugin-pseudo/src/index.ts).
