Each Atomic Props of Styli supports CSS pseudo-classes. The format of the Atomic Props that enable Pseudo-classes style is: `{prop}--{pseudo}`, for example: `bgRed400--hover`, `bgYellow500--active`, `opacity-70--disabled`, `w-20--visited`, etc.

## Hover

Use `--hover` postfix to apply hover style.

```jsx live
<Button bgRed300 bgRed400--hover red800 p4 rounded-10>
  Hover me
</Button>
```

## Active

Use `--active` postfix to apply active style.

```jsx live
<Button bgYellow300 bgYellow500--active yellow800 p4 rounded-10>
  Click Me
</Button>
```

## Disabled

Use `--disabled` postfix to apply disabled style.

```jsx live
<Box toCenter>
  <Button bgRed300 red800 p4 rounded-10 mr-20>
    Normal
  </Button>
  <Button disabled bgRed300 opacity-70--disabled red800 p4 rounded-10>
    Disabled
  </Button>
</Box>
```

## Focus

Use `--focus` postfix to apply focus style.

```jsx live
<Input bgGray200 outlineNone bgWhite--focus p2 border borderRed400--focus />
```

## Child

Use `--first-child` postfix to apply first-child style.
Use `--last-child` postfix to apply last-child style.

```jsx live
<Box>
  <Box red400--first-child>One</Box>
  <Box>Two</Box>
  <Box green400--last-child>Three</Box>
</Box>
```

## Supported pseudo classes

By default, Styli supports these pseudo classes:

- active
- checked
- disabled
- enabled
- default
- empty
- focus
- focus-within
- invalid
- hover
- link
- visited
- first-child
- last-child
- after
- before
- placeholder
- selection

If you want more pseudo classes, you can use `configure` to add it:

```jsx
import { configure } from '@styli/core'

configure({
  pseudos: ['focus-visible', 'blank'], // will merge to built-in pseudos
})
```

## About file size

Each Atomic Props in Styli supports CSS pseudo-classes and 18 commonly used pseudo-classes. You may have questions about whether this will cause the code size of the Styli package to explode. The answer is no.

In traditional CSS, if each CSS class supports Pseudo-classes, a large amount of CSS code is required, because CSS is not programmable, and it can only accumulate code through enumeration.

Thanks to the flexibility of JS, Styli only needs dozens of lines of code to make all Atomic Props support Pseudo-classes.
