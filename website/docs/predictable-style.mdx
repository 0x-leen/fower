One of most cool feature of Styli is the predictable style. In traditional CSS, it's not predictable to override style.

## Traditional CSS

For example, if you have some css rule like below:

```css
.red {
  color: red;
}
.blue {
  color: blue;
}
```

If we set `className="red blue"` to a text, it will be color `blue `.

<Tabs
  defaultValue="React"
  values={[
    { label: 'React', value: 'React', },
    { label: 'Vue', value: 'Vue', },
  ]
}>
<TabItem value="React">

```jsx live
<Text className="red blue">Styli</Text>
```

</TabItem>
<TabItem value="Vue">

```html
<span className="red blue">Styli</span>
```

</TabItem>
</Tabs>

If we reverse it to `className="blue red"` to a text, it will be color `blue `, too.

<Tabs
  defaultValue="React"
  values={[
    { label: 'React', value: 'React', },
    { label: 'Vue', value: 'Vue', },
  ]
}>
<TabItem value="React">

```jsx live
<Text className="red blue">Styli</Text>
```

</TabItem>
<TabItem value="Vue">

```html
<Text className="red blue">Styli</Text>
```

</TabItem>
</Tabs>

The color of text is not depended by the order of classNames, it's not predictable.

## In Styli

In Styli, the style is predictable. The style is depended by the order of Atomic props.

If we set props to `red400 green400`, the color of text is `green400`:

## Useful use case

When we build a reusable component, it's very useful. We can override component style elegantly.

```jsx live
function App() {
  const MyButton = (props) => {
    return <Box as="button" bgAmber500 p2 white {...props} />
  }

  const Button = styled(MyButton)

  return (
    <Box>
      <Button>default</Button>
      <Button bgGreen500 p3>
        bgGreen500
      </Button>
      <Button bgPurple500 p4>
        bgPurple500
      </Button>
      <Button bgPink500 p5>
        bgPink500
      </Button>
    </Box>
  )
}
```
