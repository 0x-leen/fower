## 什么是原子属性 (Atomic Props)

原子样式

## 开发效率更快

传统的样式开发，编写样式是你需要在 HTML 和 CSS 之间来回跳转，这很浪费时间，并且开发体验很差。使用 Styli 时，样式是直接写在模板上的，元素的样式编写会变得更加轻松、直观，这样你会更专注于更重要的东西 —— 编写样式。并且，你会发现开发体验也会高一个一个档次。

思维不用在 HMLL 和 CSS 来回切换

## 不再为命名而苦恼

名很难，你要给每一个元素命名，这是一件浪费时间并且非常痛苦的事情，实际上很多元素的类名是无关紧要的。

如果使用模板，则给文件起一个有意义的名称就足够了。 我认为，限制自己通过语义类名称来样式化事情会适得其反。

Our philosophy is that visual styling should be decoupled from content
