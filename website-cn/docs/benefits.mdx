## 局部样式

传统的 CSS 类是全局的，很容易出现样式互相影响的问题，特别多人合作开发的时候。Styli 使用原子属性编写样式，样子属性最终生成唯一的原子类名，样式只作用当前的 HTML 元素(markup)。

## 消除死代码

传统的 CSS，即使使用 OOCSS、SMACSS、BEM 等方式管理，随着项目的增大的，无用代码 CSS 代码会越来越多。原因很多，比如结构(markup)已删除，CSS 忘记删除，或者不敢删除；又比如你需要修改别人的代码，因为不熟悉，很容易滋生无用 CSS。使用 Styli 是，样式是“内联”在结构中的，基本不会出现 Dead CSS code。

## 良好的开发体验

- Styli 是 Type-Safe 的，这位你带来良好的只能提示
- 样式和结构是内聚的，易于删除、修改、复制，让你更有信心重构代码
- 编写样式是你不需要在 HTML 和 CSS 之间来回跳转，减少心烦

## 动态样式

使用 Styli，样式代码处于 JS 的上下文(context)中，你可以使用 JS 的变量、条件、函数等所有变成特性，让动态样式变得非常容易。

## 复用，不会膨胀

使用原子属性，让样式复用变得简单，大部分情况，你都不需要写新的 CSS，这样你样式代码不会随着项目的发展而逐渐膨胀。
