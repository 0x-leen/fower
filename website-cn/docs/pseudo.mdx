Styli 的每个原子属性(Atomic Props) 都支持 CSS 伪类。

## 用法

您可以直接使用 `{prop}--{pseudo}` 形式的原子属性开启伪类。

```jsx live
<Wrapper>
  <View center s-100 bgBlue30 red40 red60--hover>
    1
  </View>
  <View center s-100 bgBlue30 c="red40" c--hover="red60">
    2
  </View>
</Wrapper>
```

## Hover

```jsx live
<Wrapper>
  <Button p4 red80 bgRed30 rounded-10 outlineNone bgRed40--hover c--hover="yellow">
    Hover me
  </Button>
</Wrapper>
```

## Active

```jsx live
<Wrapper>
  <Button bgYellow30 bgYellow50--active yellow80 p4 rounded-10 outlineNone>
    Click Me
  </Button>
</Wrapper>
```

## Disabled

```jsx live
<Wrapper>
  <Button bgRed30 red80 p4 rounded-10 outlineNone>
    Normal
  </Button>
  <Button disabled bgRed30 opacity-70--disabled red80 p4 rounded-10 border="none">
    Disabled
  </Button>
</Wrapper>
```

## Focus

```jsx live
<Wrapper>
  <Input
    p2
    bgGray20
    outlineNone
    bgWhite--focus
    border="1px solid #aaa"
    border--focus="1px solid deeppink"
  />
</Wrapper>
```

## Child

```jsx live
<Wrapper>
  <View red40--first-child>One</View>
  <View>Two</View>
  <View green40--last-child>Three</View>
</Wrapper>
```

## 支持的伪类

Styli 支持下面所有伪类：

- active
- checked
- disabled
- enabled
- default
- empty
- focus
- focus-within
- invalid
- hover
- link
- visited
- first-child
- last-child
- after
- before
- placeholder
- selection

## 关于体积

Styli 每个原子属性(Atomic Props) 都支持 CSS 伪类，并且支持了 18 常用的伪类。你可能会有疑问，这会不会导致 Styli 包的代码体积爆炸，答案是不会。传统的的 CSS 方案，如果要让每个 CSS 类支持伪类，需要大量的 CSS 代码，因为 CSS 不具备编程性，它只能通过枚举的方式堆积代码。得益于 JS 的灵活性，Styli 只需几十行代码即可让所以原子属性支持伪类。如果你对如何实现伪类的插件有兴趣，可以看看[styli-plugin-pseudo](https://github.com/forsigner/styli/blob/master/packages/styli-plugin-pseudo/src/index.ts)源码。
