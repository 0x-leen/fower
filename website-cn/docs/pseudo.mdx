Styli 每个原子属性(Atomic Props) 都支持 CSS 伪类，开启伪类样式的原子属性格式为：{prop}--{pseudo}，例如：bgRed40--hover、bgYellow50--active、opacity-70--disabled、w-20--visited 等。

## Hover

```jsx live
<Button bgRed30 bgRed40--hover red80 p4 rounded-10 border="none">
  Hover me
</Button>
```

## Active

```jsx live
<Button bgYellow30 bgYellow50--active yellow80 p4 rounded-10 border="none">
  Click Me
</Button>
```

## Disabled

```jsx live
<View center>
  <Button bgRed30 red80 p4 rounded-10 border="none" mr-20>
    Normal
  </Button>
  <Button disabled bgRed30 opacity-70--disabled red80 p4 rounded-10 border="none">
    Disabled
  </Button>
</View>
```

## Focus

```jsx live
<Input
  bgGray20
  css={{ outline: 'none' }}
  bgWhite--focus
  p2
  border="1px solid #aaa"
  border--focus="1px solid deeppink"
/>
```

## Child

```jsx live
<View>
  <View red40--first-child>One</View>
  <View>Two</View>
  <View green40--last-child>Three</View>
</View>
```

## 支持的伪类

Styli 支持下面所有伪类：

- active
- checked
- disabled
- enabled
- default
- empty
- focus
- focus-within
- invalid
- hover
- link
- visited
- first-child
- last-child
- after
- before
- placeholder
- selection

## 关于体积

Styli 每个原子属性(Atomic Props) 都支持 CSS 伪类，并且支持了 18 常用的伪类。你可能会有疑问，这会不会导致 Styli 包的代码体积爆炸，答案是不会。传统的的 CSS 方案，如果要让每个 CSS 类支持伪类，需要大量的 CSS 代码，因为 CSS 不具备编程性，它只能通过枚举的方式堆积代码。得益于 JS 的灵活性，Styli 只需几十行代码即可让所以原子属性支持伪类。如果你对如何实现伪类的插件有兴趣，可以看看[styli-plugin-pseudo](https://github.com/forsigner/styli/blob/master/packages/styli-plugin-pseudo/src/index.ts)源码。
