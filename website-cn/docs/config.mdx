为了降低使用成本，我们在对应的框架包(如: @styli/react)预先进行了配置。

## 配置项

| 配置项  | 类型                                                                                                | 默认值 | 意义                                                     |
| ------- | --------------------------------------------------------------------------------------------------- | ------ | -------------------------------------------------------- |
| unit    | string                                                                                              | px     | 单位自动补全。如 Styli 会像 'm-10' 解析为 'margin: 10px' |
| inline  | boolean                                                                                             | false  | 使用内联 style 还是外部 css                              |
| prefix  | string                                                                                              | -      | 外部 css 模式下，生成的 className 前缀                   |
| theme   | Theme                                                                                               | {}     | 设计系统主题配置                                         |
| plugins | [StyliPlugin\[\]](https://github.com/forsigner/styli/blob/master/packages/types/src/StyliPlugin.ts) | []     | [styli 插件列表](plugin)                                 |

## 查看

使用 **styli.getConfig** API 可以查看单个或所有配置项

```ts
// 获取插件列表
styli.getConfig('plugins')

// 获取所有配置
styli.getConfig()
```

## 修改

**styli.configure** API 可以替换或合并配置项。

### 替换

如果您想对 Styli 进行全新的配置，您可以使用传递对象的方式，对默认配置进行替换。

```tsx
import { styli } from '@styli/react'

styli.configure({
  unit: 'px',
  theme: {},
  plugins: [],
})
```

注意：Styli 是通过插件来匹配和解析组件上的每一个属性，所以当使用配置替换时，您需要手动注册插件。详情请查询 [插件](plugin)

### 合并

传递函数的方式，可以对新旧配置进行合并。

```tsx
import { styli } from '@styli/react'

styli.configure((prevConfig) => {
  const { theme } = prevConfig
  return {
    unit: 'px',
    theme: {
      ...theme,
      breakpoints: theme.breakpoints.concat('2000px'),
    },
  }
})
```
